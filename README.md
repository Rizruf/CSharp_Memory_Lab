# CSharp_Memory_Lab

Это практический исследовательский проект для наглядной демонстрации фундаментальных концепций управления памятью в .NET и C#.

Проект был создан в рамках моего обучения для перехода от теоретических знаний к практическому, доказательному пониманию.

## Проведенные Эксперименты

В этом проекте реализовано 3 ключевых эксперимента:

### 1. Value Types vs. Reference Types
- **Цель:** Визуально доказать разницу в поведении типов-значений и ссылочных типов при передаче их в качестве аргументов в методы.
- **Результат:** Эксперимент наглядно показывает, что `Value Types` передаются по значению (копируются), а `Reference Types` — по ссылке (копируется только адрес объекта).

### 2. Производительность: Boxing vs. Generics
- **Цель:** Сравнить производительность старой коллекции `ArrayList` и современной обобщенной `List<T>`.
- **Результат:** Эксперимент демонстрирует огромное падение производительности при использовании `ArrayList` с типами-значениями из-за постоянных операций упаковки (`Boxing`). `List<T>` работает в разы быстрее, так как избегает этих накладных расходов.

### 3. Аллокация в Large Object Heap (LOH)
- **Цель:** Показать, как выделяется память для больших объектов (>85 КБ) и как это влияет на общее потребление памяти.
- **Результат:** Эксперимент доказывает, что большие объекты попадают в LOH. Эта куча не подвергается сжатию, что может приводить к фрагментации памяти, что и было продемонстрировано ростом общего потребления памяти после каждой аллокации.

## Примененные Инженерные Практики
- **Чистая архитектура:** Логика экспериментов разделена на классы-инструменты, а `Program.cs` выступает в роли "точки входа" и сценариста.
- **Осмысленные коммиты:** История коммитов ведется по профессиональным стандартам (Conventional Commits).
- **`.gitignore`:** Проект настроен для игнорирования служебных файлов Visual Studio.
